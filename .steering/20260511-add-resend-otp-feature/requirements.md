# 認証コード再送機能の実装要求

**ステアリングディレクトリ**: `.steering/20260511-add-resend-otp-feature/`
**作成日**: 2026-05-11
**参照**: `docs/product-requirements.md` / `src/app/login/verify/page.tsx`

## 1. 背景と目的
現在、8桁の認証コード入力画面に「再送」ボタンが存在するが、機能が未実装である。
ユーザーがメールを受信できなかった場合に、画面を戻ることなく再度コードをリクエストできるようにし、UXを向上させる。

## 2. ユーザーストーリー
- ユーザーは、認証コードが届かない場合、検証画面の「再送」リンクをクリックできる。
- クリックすると、現在入力されているメールアドレス宛に新しい認証コードが送信される。
- 送信中はローディング状態が表示され、送信完了後は「再送しました」という通知が表示される。

## 3. 受け入れ条件
- [ ] 「再送」ボタンをクリックした際に、実際に Supabase Auth を通じて OTP が再送されること。
- [ ] 再送処理中にボタンが二重押しされないよう無効化（Disabled）されること。
- [ ] 再送に成功した場合、ユーザーに成功メッセージが表示されること。
- [ ] 再送に失敗した場合、エラーメッセージが表示されること。
- [ ] 再送後も、引き続き同じ画面でコード入力が可能であること。

## 4. 制約事項
- セキュリティのため、短時間での連続再送を制限する仕組み（レートリミット）を考慮する（今回は Supabase 側の標準制限に準拠）。
- 既存のログインフローを壊さないこと。
